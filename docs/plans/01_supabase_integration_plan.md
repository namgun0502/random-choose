# 01. Supabase 연동 및 참여자 자동 불러오기 구현 계획서

본 계획서는 `cannon-game` 앱을 Supabase와 연동하여, 사전에 등록된 참여자 목록을 자동으로 가져오고, 게임 내에서 임시로 인원을 제외(삭제)할 수 있는 기능을 구현하기 위한 계획서입니다.

---

## 사용자 검토 필요 (User Review Required)

- **Supabase 연동 자격 증명 (URL & API Key) 관리**
  - 사용자가 편리하게 연동할 수 있도록 웹 화면에 **"Supabase 설정 모달"**을 제공하여 URL과 Anon Key를 브라우저 로컬 저장소(localStorage)에 저장하도록 구현합니다.
  - 이렇게 하면 매번 코드를 수정하지 않아도 브라우저 화면에서 쉽게 자신의 Supabase 프로젝트를 연결할 수 있습니다.

- **보안 및 암호화 관련 (규칙 8)**
  - 참여자의 이름은 일반적인 닉네임 수준이므로 기본적으로는 평문으로 저장합니다. 
  - 만약 실명이나 민감한 이름 정보를 다루셔야 해서 암호화가 필요한 경우, 향후 양방향 암호화 로직(예: CryptoJS 라이브러리를 활용한 브라우저 기준 암호화)을 추가로 적용할 수 있습니다. 이번 단계에서는 기본적인 평문 저장 및 RLS(Row Level Security) 설정이 제안됩니다.

---

## Proposed Changes (변경 예정 사항)

### 1. 데이터베이스 구성 (Supabase Migrations)

#### [NEW] [001_create_members_table.sql](file:///c:/Users/user/Desktop/cannon-game/supabase/migrations/001_create_members_table.sql)
- 참여자 이름을 저장할 `members` 테이블을 생성하는 SQL 쿼리입니다. (규칙 7에 따라 `supabase/migrations/` 폴더에 번호를 매겨 생성)

```sql
-- 1. 참여자 정보를 저장할 테이블 생성
CREATE TABLE IF NOT EXISTS public.members (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

-- 2. RLS(Row Level Security) 설정 활성화
ALTER TABLE public.members ENABLE ROW LEVEL SECURITY;

-- 3. 누구나 조회 및 삽입할 수 있는 간단한 정책 추가 (익명 사용자용)
CREATE POLICY "Allow public read access" ON public.members 
    FOR SELECT USING (true);

CREATE POLICY "Allow public insert access" ON public.members 
    FOR INSERT WITH CHECK (true);

CREATE POLICY "Allow public delete access" ON public.members 
    FOR DELETE USING (true);
```

## Verification Plan (검증 계획)

### Manual Verification
1. **Supabase 연동 테스트**:
   - 화면 우측 상단의 "Supabase 설정"을 클릭하고 자신의 Supabase 프로젝트 URL 및 Anon Key를 입력하여 저장합니다.
   - 연동 상태 표시등이 초록색(연동 성공)으로 변하는지 확인합니다.
2. **데이터 불러오기 테스트**:
   - Supabase 테이블(`members`)에 이름(예: `남건`, `홍길동`, `이순신`)을 임의로 입력해 둡니다.
   - 앱 준비 화면에서 "Supabase에서 불러오기" 버튼을 누르거나 페이지 로딩 시 자동으로 이름을 성공적으로 긁어오는지 확인합니다.
3. **임시 삭제 테스트**:
   - 목록에서 특정 이름의 `X` 버튼을 눌렀을 때, 로컬 화면에서만 지워지고 실제 Supabase 데이터베이스에는 삭제가 되지 않았는지 Supabase 대시보드에서 직접 확인합니다.
   - "처음으로" 혹은 "불러오기" 버튼을 다시 누르면 Supabase에 저장된 원본 멤버들이 다시 화면에 온전히 복구되는지 검증합니다.
